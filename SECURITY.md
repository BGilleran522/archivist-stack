# Security policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting flow:

https://github.com/BGilleran522/archivist-stack/security/advisories/new

Include the affected file or component, reproduction steps, potential impact,
and any suggested mitigation. Reports will be acknowledged and evaluated before
details are disclosed publicly.

## Supported versions

Until the project reaches 1.0, security fixes are applied to the latest commit
on `main`. Consumers should pin a commit or release and review updates before
deploying them.
